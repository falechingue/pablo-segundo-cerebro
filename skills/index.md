# index.md — Skills

Catálogo prático das skills disponíveis.

## Segundo cérebro

### `cerebro`

Finalidade: carregar mapa, contexto vivo, pessoas, projetos e pendências.

Quando usar: antes de análise importante ou quando pedir status.

Exemplo: `@pablo cerebro`

### `salve`

Finalidade: fechar sessão e propagar memória para arquivos corretos.

Quando usar: depois de decisões, reuniões, auditorias ou tarefas longas.

Exemplo: `@pablo salve`

### `relatorio`

Finalidade: gerar relatório executivo, radar de mercado, demandas, leads e projetos.

Quando usar: quando Pablo/Daiene quiserem visão consolidada.

Exemplo: `@pablo relatorio demandas`

### `retrieval-reflex`

Finalidade: política do Gbrain para preferir busca/retrieval antes de carregar arquivos grandes.

Quando usar: automático quando houver contexto grande.

Exemplo: buscar antes de abrir `_curso/transcricao-completa.md`.

## Starter Kit

### `onboarding-checklist`

Finalidade: jornada mestre do Starter Kit.

Quando usar: apenas se quiser refazer onboarding do curso.

Exemplo: `continuar jornada`

### Wizards

Inclui `wizard-agente`, `wizard-aluno`, `wizard-autonomia`, `wizard-workspace`, `wizard-conectar`, `wizard-whisper-quick`, `wizard-whatsapp`.

Quando usar: configuração guiada específica.

Exemplo: `wizard-conectar` para conexão GitHub/Search/Chromium.

### `primeira-vitoria` e `continuar-jornada`

Finalidade: gerar primeiro artefato e continuar curso.

Quando usar: quando quiser seguir o fluxo didático do Starter Kit.

## Planejamento

### `brainstorming`

Finalidade: explorar ideia antes de executar.

Exemplo: `vamos fazer brainstorming sobre um produto de BPO`.

### `writing-plans`

Finalidade: transformar ideia em plano.

Exemplo: `crie um plano para configurar GitHub do segundo cérebro`.

### `executing-plans`

Finalidade: executar plano passo a passo.

Exemplo: `execute o PLAN.md`.

### `verification-before-completion`

Finalidade: verificar evidências antes de dizer que está pronto.

Exemplo: `verifique antes de concluir`.

## Operacional

### `backup-workspace-github`

Finalidade: configurar backup GitHub.

Quando usar: depois de criar repo/token.

### `commit-diario-workspace`

Finalidade: commit diário do workspace.

Quando usar: depois de GitHub configurado.

### `cron-resume-wizards`

Finalidade: retomar wizards por cron.

Quando usar: raramente; só quando jornada guiada estiver ativa.

### `seguranca-checklist`

Finalidade: checklist de segurança.

Quando usar: antes de integrações, crons ou exposição externa.

## Canais

Quando usar: quando Pablo autorizar conexão externa.
